## What this repo is

`molcfg` is a configuration library for the molcrafts ecosystem: layered loading
from dicts / JSON / TOML / YAML / env / CLI, deep-merge with isolated copies,
recursive schema validation, source tracking via `Config.meta()`, a small registry
for resolving config strings into classes, thread-safe wrapper, POSIX file lock,
and `${path}` / `${env:VAR}` interpolation. Runtime depends only on `pyyaml`.

## Where things live

- Source code: `src/molcfg/` — flat package (config, source, merge, validation,
  registry, concurrency, errors)
- Tests: `tests/` — one `test_<module>.py` per source module plus `test_integration.py`
- Public documentation: `docs/` (zensical static site)
- Passive internal context (notes, decisions, debt, blueprint):
  `.claude/.agent/`
- Active runtime specs (alive, deleted on completion):

- Claude Code runtime config: relies on the user-level `mol` plugin; no
  project-local skills/agents/hooks/settings yet

## Architecture

Single-package, flat module layout under `src/molcfg/`. Modules are layered by
concern, not by technical layer:

- `errors` — exception hierarchy (no internal deps)
- `source` — `Source` protocol and concrete loaders (Dict, Env, Cli, Json/Toml/YamlFile)
- `merge` — `MergeStrategy`, `merge()`, `ConfigLoader`, `ProfileLoader`
- `validation` — schema validation, constraints (`Range`, `OneOf`, …), `Build`
- `registry` — `Registry` for resolving strings into classes/instances
- `concurrency` — `FileLock`, `ThreadSafeConfig`, `interpolate`
- `config` — `Config` container (attribute/path access, freeze, snapshot, meta)

Public surface is re-exported from `molcfg/__init__.py`. The package keeps zero
runtime deps beyond `pyyaml` — do not add transitive dependencies casually.

## What must never change casually

- **Runtime deps**: only `pyyaml`. Do not add others without an explicit decision.
- **Public API** in `molcfg/__init__.py`: this is a stable library. Renames or
  removals are breaking changes — schedule for a major release and update
  `CHANGELOG.md`.
- **Source-tracking contract**: every value loaded must round-trip through
  `Config.meta()` with `source` and `history`. Tests in `test_integration.py`
  protect this — do not weaken them.
- **Immutability of merge results**: `merge()` and the `MergeStrategy` family
  must always return isolated copies. No in-place mutation of inputs.
- **Python floor**: `>=3.12`. The code uses 3.12 syntax (`tomllib`, `type X = …`).
